Почему FastZapret быстрее?

Архитектура переписана с нуля. Никаких узких мест в виде одного потока обработки пакетов.

N параллельных хэндлов

WinDivert открывается по числу ядер CPU. У zapret — один поток. Результат: ×8 пропускной способности на 8-ядернике.

Batched I/O

RecvEx / SendEx читают и отправляют до 64 пакетов за один syscall. zapret делает по одному. Экономия ядра — в разы.

Zero-alloc разбор пакета

Никаких структур и копий — только индексы в исходный буфер. GC не успевает проснуться.

Узкий фильтр в драйвере

Нерелевантный трафик не покидает kernel-mode. Pass-through для не-TLS пакетов отдаёт их обратно без участия user-space.

Свой пересчёт checksum

Инлайновая реализация L4 checksum без вызовов в WinDivertHelper. Сэкономили один syscall на каждый пакет.

Hostname-aware стратегии

SNI распознаётся за наносекунды — под каждый сервис применяется своя оптимальная техника обхода.

Что разблокирует

Четыре сервиса из коробки. Каждый со своим оптимальным пайплайном.

YT

YouTube

HTTPS + QUIC. Видео без буферизации.

DC

Discord

Текст и голос. UDP-каналы тоже работают.

TG

Telegram

MTProto через 443. Без прокси.

RB

Roblox

HTTPS вход + UDP игровой трафик.
